# すなっちゃん
## 前提
### プラットフォーム概要
- 名称：シコウラボ
- 意味：思考／志向／試行 × ラボ
- 位置づけ：専門家の思考・研究プロセスを一般ユーザーが覗ける場
**懸念① チャット治安**
- AI荒らし検出：実装可（初期は検知・通報補助）
- 新規ユーザー権限制限：実装容易（閲覧／質問のみ）
- チャット分割：実装可（招待制・上位プラン限定）
→ 権限制御＋場の分離で対処可能
**懸念② チャットがノイズ**
- チャットON/OFF：実装容易
- 上位プラン限定チャット：実装可
→ 記事中心、チャットは任意・上位向け
**チャットの位置づけ**
- 主：記事・思考の可視化
- 副：思考過程を覗く補助機能
- 全員参加型にはしない
**開発フェーズ**
- Phase1：記事、限定チャット、権限制御
- Phase2：ルーム分割、信頼度制御
- Phase3：AI荒らし検出強化
**総括**
- 技術的に実装困難な機能なし
- 成否は実装より運用設計に依存
## 議事録
- これまでの目的、Discordで提供しているサービスを見やすくしたい
- これからの目的、チャット機能自体プラットフォームに必要ないと結論づいた
    - じゃあDiscordもいらない？
- 真の目的は「記事を作って発信すること」と「他のプラットフォームで得られない情報を初心者でもわかりやすく簡潔に伝える」
- なぜチャットがいらないのか
    - そもそもアンケートをとった際に、アクティブで会話しているユーザーが15人程度(1000人中)
    - 98.5%がROM専となっている
    - 全てのユーザーにとって全体利益になるようにしたい
        - 今のチャットルームもギャンブルじみていると感じる(管理者視点)。根拠を伴わず博打の賭けになってしまっている。
        - プラットフォームのメイン機能として実装するとこれらを含めてノイズになる
        - 話さないとって居づらいし不要
    - プラットフォームの高級感、専門性を維持したい
- ユーザーの目的は、プロの知見が欲しい、専門的な情報が欲しい←これらのユーザーの満足度を下げる要因になる
- 記事に当てるリースをチャットに当てる、20人しか使用していないチャットに当てるのは非効率である
- なぜチャットがいらないのかまとめ
    - 98.5％のROM専を優先したい(全体利益を優先)
        - 読むことだけに価値を置いている人を守る
    - ブランドの高級感や専門性を維持するため
        - 誰でも書き込めると掲示板のようになって、プロの意識を持つ人の満足度を下げてしまう懸念
    - 運営コストの効率化
        - 記事の執筆に当てるリソースをチャットに当てない
- Didscordの扱いについて
    - 上位プランを設けてプラットフォームとは別で提供する
- プラットフォームについて
    - 基本的に受動的でよい
- ロールはどうする？
    - 管理者
    - 一般ユーザー(記事投稿)
    - 一般ユーザー
- プラン
    - ベースの記事は一般プランでもできる
    - プレミアムの記事はプレミアムプランしか読めない

## 実装機能
- 記事とニュースの投稿機能
    - 管理者用と一部の選ばれたユーザーが投稿できる
    - 記事: 筆者の意見があるもの
    - ニュース: 実態があるもの
        - RSSかAPIかで取得する
        - 日本語訳したい(旨み)
            - DeepL, LLM, 人間
- 個別銘柄の表示機能
    - 検索機能
    - 板とか速度が求められるものは重視しなくて良い
    - チャートのビジュアル化、値動き、応用が効いた内容を初心者でもわかりやすいように表現する
    - どんな機能を盛り込むか相談したい
    - ユーザーに対しての価値提供: 大手にない投資判断材料を簡潔なUIでの提供
        - テクニカルな情報も簡潔に伝える
        - 大手にプラスアルファの役割を担う
    - 表示内容
        - MUST
            - 株価のチャート: 一般的なやつ
            - 株のファンダメンタルズ
            - UnusualOptionActivity: 一億ドルとかの取引を表示する
            - 関連する記事とニュース: このアプリで表示されている記事とニュース
            - 証券会社の格付け評価: 個別銘柄に対して買いか保有か売りかとか目標株価とか
        - WANT
            - 株価チャートにニュースを表示する: N％以上だけのニュースを表示するようにする(閾値はユーザーに委任)
- アンケート機能
    - 邪魔に思われない工夫がいる
    - 予約投稿(WANT)
    - 選択肢は9つまで許容
    - アンケートのカテゴリで絞り込む
        - 5つくらいかな？少なめ
- アナリティクス(want)
    - SQLをグラファナで表現する
    - 記事のリード件数のカラムやこの機能が使われたなどの件数を保持するカラムや仕組みが必要
    - どういったアナリティクスが欲しいか
        - 記事とニュースのインサイト
        - ピックされた銘柄
        - 入会退会者数
