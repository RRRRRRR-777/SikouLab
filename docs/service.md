## 3. コンセプト TBD

### サービスの核心価値

**「専門家の記事を購読する」ことがこのサービスの売り**

- ユーザーは記事を読むためにサブスクリプションに加入する
- 記事の質と継続的な発信がサービス価値の源泉
- 銘柄情報やニュースは記事価値を補完する付加機能

### 記事購読型コミュニティとして

- 専門家の知見・思考プロセスを、初心者にも分かりやすく簡潔に提供
- ROM専ユーザーを前提とした「読むだけで価値がある体験」を最優先
- 他プラットフォームでは得られない投資判断材料を、整理されたUIで提供

## 6. サービス品質

### 6.1 パフォーマンス

**方針:** リリース前に Lighthouse CI で劣化検知、リリース後に Cloud Monitoring で継続監視

| 指標 | 目標値 | 計測タイミング | 計測ツール |
|---|---|---|---|
| ページ表示速度（LCP） | 2.5秒以内 | CI（リリース前） + 本番（継続） | Lighthouse CI + Cloud Monitoring |
| APIレスポンスタイム（p95） | 500ms以内 | 本番（継続） | Cloud Monitoring |

| 項目 | Lighthouse CI | Cloud Monitoring |
|---|---|---|
| **コスト** | 無料 | GCP含む |
| **タイミング** | PR/マージ時 | 継続 |
| **対象** | フロントエンド（LCP, FCP, CLS等） | API応答時間、レイテンシ |
| **閾値** | LCP 2.5秒未満でCI失敗または警告 | 閾値超過時にアラート |
| **結果** | PRコメントに出力 | Cloud Monitoring ダッシュボード |

※ 詳細な計測フロー・アラート設定は [ADR-010: 品質計測](./adr/010-quality-metrics.md) を参照

### 6.2 可用性（SLA）

| 指標 | 目標値 | 計測ツール |
|---|---|---|
| サービス稼働率 | 99.5% | Cloud Monitoring Uptime Check |
| メンテナンスウィンドウ | なし | - |

**設定:** チェック間隔1分、タイムアウト10秒、失敗判定「2回連続」、対象 `/health` エンドポイント

※ アラート設定・通知方法は [ADR-007: ログ監視](./adr/007-log-monitoring.md) を参照

### 6.3 品質（リグレッション防止）

**目的:** ユーザーにとって「新機能が使える」より「今まで使えていた機能が使えなくなる」方が影響が大きいため、リグレッションをゼロにすることを目指す。

| 指標 | 目標値 | 記録場所 | 可視化 |
|---|---|---|---|
| 市場障害数 | 0件/1リリース | Jira（ラベル: market-defect, regression） | Metabase |
| ロールバック回数 | 0回/1リリース | Jira（ラベル: rollback） | Metabase |

**用語定義:**
- **市場障害**: リリース後に本番環境で発見された障害
- **リグレッション**: 以前は正常に動作していた機能が、変更により動作しなくなること

※ Jira運用手順・障害対応フローは [ADR-010: 品質計測](./adr/010-quality-metrics.md) を参照

### 6.4 セキュリティ

| 項目 | 方針 |
|---|---|
| **通信暗号化** | 全通信をTLS 1.2以上で暗号化（HTTPS必須）。Cloud Runで強制、SSL Labsで定期確認 |
| **認証・認可** | OAuth 2.0（Google / Apple / X）による認証、ロールベースの権限制御。認証ログをCloud Loggingで監視 |
| **データ保護** | 個人情報の暗号化保存、Cloud Audit Logsで記録・保持（7日） |

※ 詳細な監視設定は [ADR-007: ログ監視](./adr/007-log-monitoring.md) を参照

### 6.6 品質メトリクスのサマリ

| カテゴリ | 計測ツール | タイミング | コスト |
|---|---|---|---|
| パフォーマンス（FE） | Lighthouse CI | PR/マージ時 | 無料 |
| パフォーマンス（全体） | Cloud Monitoring | 継続 | GCP含む |
| 可用性 | Cloud Monitoring | 継続 | GCP含む |
| リグレッション | Jira + Metabase | リリースごと | 無料 |
| セキュリティ | Cloud Logging | 継続 | GCP含む |

**定期レビュー:** リリースごとに品質メトリクスをレビュー。市場障害発生時は根本原因分析（RCA）を実施
