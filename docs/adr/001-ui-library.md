# ADR-001: UIライブラリの選定

## ステータス
承認済み

## コンテキスト
フロントエンド開発において、UIコンポーネントライブラリとCSSフレームワークの選定が必要。
デザイナーがPencilで作成したデザインに追従して実装する運用のため、デザインカスタマイズの自由度が重要。

## 決定
**shadcn/ui + Tailwind CSS** を採用する。

## 検討した選択肢

| ライブラリ | カスタマイズ自由度 | 学習コスト | デザインツール連携 |
|-----------|-------------------|-----------|----------|
| **shadcn/ui** | ◎ | ○ | ○ |
| Radix UI | ◎ | △ | △ |
| Headless UI | ◎ | △ | △ |
| Chakra UI | ○ | ◎ | ○ |
| MUI | △ | ○ | ◎ |
| Mantine | ○ | ○ | △ |

## 理由
1. **カスタマイズ自由度が高い**: コピペ方式でコンポーネントを取り込み、Tailwindで自由に編集可能
2. **デザインツール連携**: Pencil MCPと組み合わせることで、デザイン→実装の効率化が期待できる
3. **チームの経験**: 既に使用経験があり、学習コストが低い

## 影響
- デザイナーとコンポーネント名・デザイントークンを統一する
- Tailwind CSSの設定でデザイントークン（色、スペーシング等）を定義する
