# ADR-001: UIライブラリの選定

## ステータス
承認済み

## コンテキスト
フロントエンド開発において、UIコンポーネントライブラリとCSSフレームワークの選定が必要。
デザイナーがFigmaで作成したデザインに追従して実装する運用のため、デザインカスタマイズの自由度が重要。

## 決定
**shadcn/ui + Tailwind CSS + Figma Kit** を採用する。

## 検討した選択肢

| ライブラリ | カスタマイズ自由度 | 学習コスト | Figma連携 |
|-----------|-------------------|-----------|----------|
| **shadcn/ui** | ◎ | ○ | ○ 公式Figma Kit |
| Radix UI | ◎ | △ | △ |
| Headless UI | ◎ | △ | △ |
| Chakra UI | ○ | ◎ | ○ |
| MUI | △ | ○ | ◎ |
| Mantine | ○ | ○ | △ |

## 理由
1. **カスタマイズ自由度が高い**: コピペ方式でコンポーネントを取り込み、Tailwindで自由に編集可能
2. **Figma Kitの存在**: デザイナーと開発者が同じコンポーネント名・デザイントークンを共有できる
3. **チームの経験**: 既に使用経験があり、学習コストが低い
4. **Figma MCP連携**: プロジェクトで設定済みのFigma MCPと組み合わせることで、デザイン→実装の効率化が期待できる

## 影響
- Figma Kitをデザイナーと共有し、コンポーネント名・デザイントークンを統一する
- Tailwind CSSの設定でデザイントークン（色、スペーシング等）を定義する
