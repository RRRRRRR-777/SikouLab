# ADR-002: フォーム管理ライブラリの選定

## ステータス
承認済み

## コンテキスト
フロントエンドでのフォーム管理（状態管理、バリデーション、送信処理）のためのライブラリ選定が必要。

プロジェクトで想定されるフォーム：
- ログイン: OAuthボタンのみ（フォーム不要）
- アンケート投票: 選択肢を選ぶ、送信ボタン（シンプル）
- 記事作成・編集: タイトル、Markdownエディタ、カテゴリ、公開日時（中程度）
- プロフィール編集: 表示名、通知設定など（シンプル）
- 管理画面: 検索フィルター、ロール変更など（中程度）

「受動的体験を最優先」「能動操作は最小」という方針から、フォームは比較的シンプル。

## 決定
**React Hook Form + Zod** を採用する。

## 理由
1. **プロジェクトのフォーム複雑度に適合**: シンプル〜中程度のフォームに対応可能
2. **型安全なバリデーション**: ZodスキーマからTypeScript型を推論できる
3. **パフォーマンス**: React Hook Formは不要な再レンダリングを防ぐ設計
4. **エコシステム**: 広く採用されており、ドキュメント・事例が豊富

## 影響
- Zodスキーマを定義してバリデーションルールを一元管理する
- 記事作成のMarkdownエディタは別途専用ライブラリを検討する
