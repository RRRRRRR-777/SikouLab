# Test Cases Skill

## 概要

詳細設計書（`docs/functions/{category}/{name}.md`）の「テストケース」セクションを作成するSkillです。
境界値分析・同値分割・デシジョンテーブル・状態遷移テストの手法を用いて、網羅的かつ実用的なテストケースを定義します。

## 原則

- **網羅性**: 正常系・異常系・境界値・分岐網羅をカバーする
- **実用性**: 実装・保守が可能な範囲に留める（無限の組み合わせは避ける）
- **明確性**: 前提条件・入力・期待結果が明確に記載されている

## 必須入力

このSkillを呼び出す際は、以下の情報を提供してください：

- 機能名
- 機能要件（`docs/functions/{category}/{name}.md` の機能要件セクション）
- 関連するシーケンス図・フロー図
- 入力パラメータ・バリデーションルール
- ステータス遷移がある場合はその定義

## 出力テンプレート

```markdown
## テストケース

### 単体テスト

| テスト項目 | 対応仕様 | 観点 | 期待値 |
|------------|----------|------|--------|
| {項目1} | 機能要件1/機能仕様1 | 正常系 | {期待値} |
| {項目2} | 機能要件1/機能仕様1 | 異常系 | {期待値} |
| {項目3} | 機能要件1/機能仕様1 | 境界値 | {期待値} |
```

## テスト設計手法

### 1. 境界値分析（必須）

入力値の境界とその前後を必ずテストする。

| データ型 | テストすべき境界値 |
|---------|-------------------|
| **数値** | 最小値-1, 最小値, 最小値+1, 最大値-1, 最大値, 最大値+1, 0, -1 |
| **文字列長** | 0文字, 1文字, 最大長-1, 最大長, 最大長+1 |
| **配列・リスト** | 空, 1要素, 最大要素数-1, 最大要素数, 最大要素数+1 |
| **日時** | 最小日付-1日, 最小日付, 最大日付, 最大日付+1日, 閏日, 月末・月初 |
| **ページネーション** | page=0, page=1, page=最終ページ, page=最終ページ+1, limit=0, limit=最大 |
| **金額** | 0, 1, 最小単位-1, 最小単位, 最大額-1, 最大額, 負数 |

**ページネーションの例**:
```markdown
| TC-PAG-01 | 機能要件1/機能仕様1 | 境界値 | page=0 で400エラー |
| TC-PAG-02 | 機能要件1/機能仕様1 | 境界値 | page=1 で正常に1ページ目取得 |
| TC-PAG-03 | 機能要件1/機能仕様1 | 境界値 | page=最終ページ で正常に最終ページ取得 |
| TC-PAG-04 | 機能要件1/機能仕様1 | 境界値 | page=最終ページ+1 で空配列返却 |
| TC-PAG-05 | 機能要件1/機能仕様1 | 境界値 | limit=0 で400エラー |
| TC-PAG-06 | 機能要件1/機能仕様1 | 境界値 | limit=1 で1件返却 |
| TC-PAG-07 | 機能要件1/機能仕様1 | 境界値 | limit=最大 で最大件数返却 |
| TC-PAG-08 | 機能要件1/機能仕様1 | 境界値 | limit=最大+1 で400エラー |
```

### 2. 同値分割

入力領域を有効・無効なグループに分け、各グループから代表値をテストする。

**例: 年齢バリデーション（18歳以上のみ有効）**
```markdown
| TC-AGE-01 | 機能要件1/機能仕様2 | 正常系 | 18歳で登録成功（有効同値クラスの最小値） |
| TC-AGE-02 | 機能要件1/機能仕様2 | 正常系 | 35歳で登録成功（有効同値クラスの代表値） |
| TC-AGE-03 | 機能要件1/機能仕様2 | 正常系 | 100歳で登録成功（有効同値クラスの最大値） |
| TC-AGE-04 | 機能要件1/機能仕様2 | 異常系 | 0歳で400エラー（無効同値クラス: 未満） |
| TC-AGE-05 | 機能要件1/機能仕様2 | 異常系 | 17歳で400エラー（無効同値クラス: 境界未満） |
| TC-AGE-06 | 機能要件1/機能仕様2 | 異常系 | -1歳で400エラー（無効同値クラス: 負数） |
| TC-AGE-07 | 機能要件1/機能仕様2 | 異常系 | null で400エラー（無効同値クラス: null） |
| TC-AGE-08 | 機能要件1/機能仕様2 | 異常系 | "abc" で400エラー（無効同値クラス: 型不一致） |
```

### 3. デシジョンテーブル（複数条件の組み合わせ）

複数の入力条件が組み合わさる場合、全ての組み合わせを網羅する。

**例: 記事公開判定（ログイン状態 × 公開フラグ × 公開日時）**
```markdown
| TC-ART-01 | 機能要件2/機能仕様1 | 正常系 | ログイン済 + 公開済 + 公開日時過去 → 記事表示 |
| TC-ART-02 | 機能要件2/機能仕様1 | 正常系 | 未ログイン + 公開済 + 公開日時過去 → 記事表示 |
| TC-ART-03 | 機能要件2/機能仕様1 | 異常系 | ログイン済 + 非公開 + 公開日時過去 → 404エラー |
| TC-ART-04 | 機能要件2/機能仕様1 | 異常系 | 未ログイン + 非公開 + 公開日時過去 → 404エラー |
| TC-ART-05 | 機能要件2/機能仕様1 | 異常系 | ログイン済 + 公開済 + 公開日時未来 → 404エラー |
| TC-ART-06 | 機能要件2/機能仕様1 | 異常系 | 未ログイン + 公開済 + 公開日時未来 → 404エラー |
```

### 4. 状態遷移テスト

ステータス変更や状態遷移を持つ機能は、全ての状態と遷移パスをテストする。

**例: サブスクリプションステータス**
```markdown
| TC-SUB-01 | 機能要件3/機能仕様1 | 正常系 | pending → active に遷移成功 |
| TC-SUB-02 | 機能要件3/機能仕様1 | 正常系 | pending → canceled に遷移成功 |
| TC-SUB-03 | 機能要件3/機能仕様1 | 正常系 | active → canceled に遷移成功 |
| TC-SUB-04 | 機能要件3/機能仕様1 | 異常系 | active → pending は遷移不可（400エラー） |
| TC-SUB-05 | 機能要件3/機能仕様1 | 異常系 | canceled → active は遷移不可（再登録必要） |
| TC-SUB-06 | 機能要件3/機能仕様1 | 異常系 | canceled → pending は遷移不可（400エラー） |
```

### 5. エッジケース・特殊値

| カテゴリ | テストすべき値 |
|---------|--------------|
| **Null・Undefined** | null, undefined, "" |
| **データ型** | 文字列で数値を渡す, 数値で文字列を渡す |
| **エンコーディング** | 日本語全角, 絵文字, 特殊文字（<>&"'）, SQLインジェクション |
| **ID・識別子** | 存在しないID, 削除されたID, 他ユーザーのID |
| **権限** | ログイン済み/未ログイン, 管理者/一般ユーザー |
| **並列性** | 同時更新, 重複リクエスト |
| **ネットワーク** | タイムアウト, 500エラー, レート制限 |

**例: ID・権限のエッジケース**
```markdown
| TC-ID-01 | 機能要件1/機能仕様3 | 異常系 | 存在しないUUIDで404エラー |
| TC-ID-02 | 機能要件1/機能仕様3 | 異常系 | 削除済みのUUIDで404エラー |
| TC-ID-03 | 機能要件1/機能仕様3 | 異常系 | 他ユーザーのUUIDで403エラー |
| TC-ID-04 | 機能要件1/機能仕様3 | 異常系 | 無効なUUID形式で400エラー |
| TC-ID-05 | 機能要件1/機能仕様3 | 異常系 | 未ログインで401エラー |
```

**例: エンコーディング・特殊文字**
```markdown
| TC-ENC-01 | 機能要件1/機能仕様4 | 正常系 | 日本語全角（漢字・ひらがな・カタカナ）で正常保存 |
| TC-ENC-02 | 機能要件1/機能仕様4 | 正常系 | 絵文字（😀）で正常保存 |
| TC-ENC-03 | 機能要件1/機能仕様4 | 正常系 | 特殊文字（<>&"'）でエスケープされて保存 |
| TC-ENC-04 | 機能要件1/機能仕様4 | 異常系 | SQLインジェクションパターン（"' OR '1'='1）で無害化 |
```

## 自己レビューチェックリスト

提示前に以下を確認すること：

- [ ] 正常系：主要な成功パスがカバーされているか
- [ ] 異常系：エラーハンドリングが網羅されているか
- [ ] 境界値：入力の端値・null・空文字が考慮されているか
- [ ] 分岐網羅：if/switch の全分岐がカバーされているか
- [ ] 状態遷移：全ての状態と遷移パスが考慮されているか（該当機能の場合）

## 実行手順

1. **機能要件の確認**: 対象機能の詳細設計書を読み込み、機能要件を把握する
2. **入力パラメータの特定**: すべての入力パラメータとバリデーションルールを特定する
3. **状態遷移の確認**: ステータス遷移がある場合はその定義を確認する
4. **テストケースの設計**:
   - 境界値分析で各パラメータの境界テストを設計
   - 同値分割で有効・無効なグループをテスト
   - デシジョンテーブルで複数条件の組み合わせをテスト
   - 状態遷移テストで全ての遷移パスをテスト
   - エッジケースで特殊値をテスト
5. **カバレッジマトリックス作成**: 自己レビューチェックリストに基づきマトリックスを作成
6. **ユーザーへの提示**: テストケース一覧とカバレッジマトリックスを提示
