# Git Commit Skill

## コミットメッセージ形式
SikouLabプロジェクトでは、簡潔で一貫性のあるコミットメッセージを推奨します。

### 基本形式
```
<type>: <subject> (#チケット番号)

<body (optional)>
```

**チケット番号**: Jiraチケット番号を `(#123)` 形式でサブジェクトの最後に記載

### Type一覧
* `feat`: 新機能追加
* `fix`: バグ修正
* `docs`: ドキュメントのみの変更
* `style`: コードの意味に影響しない変更（フォーマット、セミコロンなど）
* `refactor`: バグ修正や機能追加を伴わないコード改善
* `test`: テストの追加・修正
* `chore`: ビルドプロセスや補助ツールの変更

### Subject（件名）ルール
* 50文字以内
* 命令形で記述（例: "Add" not "Added" or "Adds"）
* 先頭は大文字
* 末尾にピリオドを付けない
* 日本語で記述

### Body（本文）ルール（任意）
* なぜこの変更が必要か（Whatではなく Why）
* 72文字で改行

### 例
```
feat: 記事予約投稿機能を追加 (#123)

ユーザーが指定した日時に記事を自動公開できるようにしました。
Stripe連携により有料プランユーザーのみ利用可能です。
```

```
fix: ログイン後のリダイレクトが失敗する問題を修正 (#456)

OAuth認証後のセッション確立前にリダイレクトが
発生していたため、セッション確立を待機するよう変更。
```

## コミット前チェックリスト
- [ ] 変更内容が単一の目的に集中している
- [ ] テストが通る（該当する場合）
- [ ] 不要なコメントアウトやデバッグコードが含まれていない
- [ ] シークレット情報（APIキー等）が含まれていない
