# 開発ワークフロー

## 基本方針

- ウォーターフォール型 + V字モデルに基づくテスト設計
- 設計先行：設計書完成後に実装開始
- TDD：テストケース先行で実装

## Skill呼び出しルール

設計ドキュメントの**新規作成・大幅修正**時は、必ず対応するskillを呼び出すこと。

| 作業 | 必須skill |
|------|-----------|
| 要件定義書の作成・修正 | `/requirements` |
| 基本設計書の作成・修正 | `/system-design` |
| 詳細設計書の作成・修正 | `/feature-spec` |
| 作業見積もりの追加 | `/feature-spec-estimate-cost` |

## 実装開始条件

- feature-spec全セクション記載済
- 単体テストケース（大枠）記載済
- バックエンド機能: openapi.yaml更新済

## タスク実行フロー

1. **Jiraチケットがない場合** → 設計書確認 → チケット構成をユーザーに提案 → 承認後に作成
2. **チケット開始時** → development-plan.md + 詳細設計書を確認、完了条件を把握
3. **実装中** → 設計書を常に参照、疑問はユーザーに質問
4. **完了前レビュー** → レビュー観点を提示、ユーザー承認後にコミット
5. **チケットクローズ** → 「やったこと」記載、ステータス更新

> 詳細手順・チェックリスト・並列実行ルールは `/workflow` skillを参照。
